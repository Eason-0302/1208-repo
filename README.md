### Git Revert 練習紀錄

我在此次作業中，先故意修改 footer 文字作為測試用 commit，
接著使用 `git revert` 將該次修改回復，讓網頁內容恢復正常。
本次練習讓我理解 `git revert` 是以新增一筆 commit 的方式進行還原，
而非直接刪除歷史紀錄。

## A/B 測試：Skills 顯示方式

為了比較不同呈現方式的可讀性與使用者體驗，本專案實作了兩個版本：

- A（進度條版）: 已合併至 `main`。特色為以進度條顯示每項技能的熟練度百分比，易於讓檢視者快速量化技能強度，適合履歷場景。

- B（標籤雲版）: 保留為 PR (`feature/skills-tags`)，未合併。特色為以標籤雲展示技能，強調關鍵字視覺層級，但相對較不精準且對於無障礙／篩選需求較不友善。

採用理由（選用 A 而非 B）：
- 進度條能清楚傳達熟練度數值，對 HR 或面試官判斷更直觀。
- 標籤雲較適合展示關鍵字但無法量化熟練度，容易造成資訊解讀上的歧義。

未採用版本仍保留為 PR，以便之後進行使用者測試或改良後再重新評估。
